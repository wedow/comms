## Summary
End-to-end test that exercises the full daemon lifecycle: start daemon with a mock Telegram provider, receive messages, verify files written to `.comms/`, verify callback fired, stop daemon via context cancellation, verify PID file cleaned up, verify polling offset persisted.

## Implementation Notes
- Dependencies: Tasks 3.1-3.9 (all Phase 3 tasks)
- This is a comprehensive integration test covering:
  1. Start daemon with mock/fake Telegram provider
  2. Receive messages through the fake provider
  3. Verify message files are written to the `.comms/` directory structure
  4. Verify callback was fired (use a callback script that touches a marker file)
  5. Stop daemon via context cancellation (not SIGTERM, since this is a test)
  6. Verify PID file is cleaned up after shutdown
  7. Verify polling offset is persisted to disk after shutdown

- The fake provider should deliver a known set of messages, then block until context is cancelled
- Callback script should be a simple shell command like `touch <marker-file>`
- Test should complete within a reasonable timeout
